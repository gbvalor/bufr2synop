Detailed Description

Array of offset in bits for every subset in a non-compressed bufr. Offset is counted in bits from the init of SEC4 data, usually b1it 32.

To set the offset for non compressed bufr it is neccesary to parse the expanded tree and extact the data of the prior subsets. If we already know the bit offset of the data then we can access directly to the subset data without parsing the prior subsets.

The utility of these offset array is that it can be write to and read from a binary file and access quickly to the desired subset data.
